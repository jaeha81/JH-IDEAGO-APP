# IDEAGO SCREEN LIST

## 문서 목적
이 문서는 IDEAGO 프론트엔드 구현 전에 필요한 화면 목록을 정리한 것이다.
여기서 "화면"은 독립 페이지, 주요 작업 화면, 필수 모달/드로어/오버레이 상태를 포함한다.
이 문서는 현재 구현 기준선이며, 이후 수정될 수 있다.

---

# 1. 메인 화면 목록

## S-01. Landing / Home
### 목적
서비스 첫 진입 화면. 사용자가 IDEAGO의 정체성을 이해하고 새 프로젝트를 시작하거나 기존 프로젝트를 여는 화면.

### 필수 요소
- IDEAGO 로고/타이틀
- (MultiGenius) 보조 표기
- 영문 설명
- 한글 설명
- New Project 버튼
- Open Existing Project 버튼
- 최근 프로젝트 진입 영역 (선택)

### 비고
- 미니멀하고 브랜드 정체성이 보여야 함
- 모바일/태블릿 모두 대응

---

## S-02. New Project Setup
### 목적
새 프로젝트 생성 전 기본 설정 화면. 프로젝트명과 에이전트 구성을 정한다.

### 필수 요소
- 프로젝트명 입력
- 프로젝트명 자동 생성 옵션
- 에이전트 수 선택
- 에이전트 추가 버튼
- 각 에이전트 역할명 입력 필드
- 시작하기 버튼

### 비고
- 사용자는 처음부터 명확하지 않을 수 있으므로 복잡한 입력은 금지
- "대충 시작해도 된다"는 성격의 안내 문구 허용

---

## S-03. Main Workspace
### 목적
IDEAGO의 핵심 작업 화면. 자유 드로잉, 글쓰기, 이미지 업로드, 에이전트 협업이 이루어지는 메인 캔버스 화면.

### 필수 요소
- 대형 캔버스 영역
- 기본 툴바
- 에이전트 결과/채팅 패널
- 업로드 버튼
- Detail View 진입 버튼
- Export 버튼
- 프로젝트명 표시
- 저장 상태 표시(선택)

### 캔버스 툴 필수 범위
- 펜
- 브러시
- 지우개
- 선택
- 이동
- 복사
- 자르기
- 텍스트
- 도형
- 색상 선택
- 확대/축소

### 비고
- 태블릿 가로 기준 핵심 생산성 화면
- 이 화면이 서비스의 중심

---

## S-04. Project Summary / History
### 목적
현재 프로젝트의 핵심 내용, 에이전트 구성, 목적 상태, 작업 요약, 다음 단계 등을 정리해서 보여주는 화면.

### 필수 요소
- 프로젝트명
- 생성일 / 수정일
- 에이전트 구성 요약
- 핵심 아이디어 요약
- 사용자 메모 요약
- 현재 목적 상태
- 다음 추천 액션
- 작업 히스토리 요약

### 비고
- 전달 전 확인용 성격도 있음

---

## S-05. Export Screen
### 목적
프로젝트를 구조화된 폴더/ZIP 패키지로 내보내기 전에 포함 내용을 확인하고 내보내는 화면.

### 필수 요소
- 내보내기 요약
- 포함 파일 목록 미리보기
- ZIP Export 버튼
- 설명 문구
- 선택 시 포함 옵션(선택)

### 포함 파일 예시
- project-summary.md
- agents.md
- canvas-data.json
- uploads/
- visualization/
- instructions.md
- export-manifest.json

---

# 2. 보조 화면 / 오버레이 / 모달 목록

## O-01. Agent Panel
### 성격
메인 Workspace 내부 보조 패널

### 목적
사용자 질의 입력 및 에이전트 요약 결과 확인

### 필수 요소
- 입력창
- 에이전트별 역할 라벨
- 요약 응답
- 더보기/상세보기
- 접기/펼치기

### 비고
- 기본은 요약형
- 전체 토론 로그를 기본 노출하지 않음

---

## O-02. Upload + Markup State
### 성격
메인 Workspace 내 상태

### 목적
업로드한 이미지/사진/도면 위에 직접 주석, 도형, 메모, 강조를 넣는 상태

### 필수 요소
- 이미지 업로드
- 캔버스 내 배치
- 상단 드로잉/마킹 가능
- 화살표/도형/텍스트/강조 표시 가능

### 비고
- 삼성 노트식 마킹 흐름 참고 가능
- 단, IDEAGO는 더 범용적이고 전문적인 구조여야 함

---

## O-03. Detail View
### 성격
선택형 모달 또는 드로어 또는 별도 패널

### 목적
사용자가 원할 때만 결과 시각화를 더 정리된 형태로 보는 뷰

### 필수 요소
- 시각화 결과 영역
- 닫기 버튼
- 원본 대비 보기(선택)
- 확대 보기(선택)

### 명칭 기본안
- Detail View

### 비고
- 항상 열려 있으면 안 됨
- 선택적으로 호출되는 구조여야 함

---

## O-04. Agent Add/Edit Modal
### 성격
모달

### 목적
프로젝트 중간에도 에이전트를 추가/수정/삭제하는 설정 창

### 필수 요소
- 에이전트 이름/역할 수정
- 추가
- 삭제
- 저장

---

## O-05. Export Confirmation Modal
### 성격
모달

### 목적
내보내기 전 최종 확인

### 필수 요소
- 내보내기 포함 항목 요약
- 확인 / 취소
- 파일 형식 안내

# 5. 화면 정의 원칙

1. UI는 바뀔 수 있지만 기능 책임은 유지한다.
2. Agent Panel은 독립 서비스가 아니라 Workspace 보조 영역이다.
3. Detail View는 필수 상시 노출 패널이 아니라 선택형 기능이다.
4. Upload + Markup은 핵심 기능이다.
5. Export는 단순 다운로드가 아니라 구조화 전달 기능이다.
